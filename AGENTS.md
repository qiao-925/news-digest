# News Digest 项目规则

## 偏好

- 保持轻量，不项目化
- 输出风格偏简洁，不要长篇大论和说教感
- 推荐内容时直接用 `Start-Process` 打开链接，不用只贴 URL
- 不需要反复确认，直接做

## 执行原则

- 执行前先做充分调研，不要拼手感
- 本机数据（书签、日志等）始终实时读取，不做本地存档/缓存（JSON 快照等），保证数据实时性
- 变化慢的数据（关注列表、追番等）可以本地化，定期同步更新
- 生成的总览文档（如分类表）可以保留，但每次操作前要重新读取源数据

## 数据安全

- 敏感数据（密钥、序列号、个人 ID）只留本地，不推远端
- 推到 GitHub 的内容需脱敏
- 本项目为 Public，用于个人项目展示

## 自检规则

- **修改后必须自检**：对关键功能修改后，添加自检验证实际效果
- **当前工具**：使用 Playwright 真实浏览器交互验证（可点击、统计实际渲染）
- **固定流程**：`修改 → 自检 → 确认输出` 三步闭环
